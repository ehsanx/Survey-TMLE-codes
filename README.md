## What this repository contains

This repository holds the **complete analysis code** for the paper, the **locked
summary-result files** every quoted number is computed from, and a **number audit**
that re-derives every figure/table/inline number from those files. Specifically:

- **`codes/`** — the estimator engine, the simulation pipeline, the eight
  enhancement/sensitivity simulations, the table/figure renderers, and the audit.
- **`Nhanes/`** — the full four-example NHANES analysis (download → build → impute →
  estimate → figures), plus the three NHANES sensitivity analyses (including the
  multiple-imputation run behind the headline Table 2).
- **`results/`** — the locked summary CSVs the paper quotes (small; committed).
- **`Writing/`** — the output folders the renderers write the LaTeX table/figure
  fragments into.

A reviewer can, from a fresh clone (after `Rscript install_packages.R`):

1. **Verify every number in the paper** — `Rscript codes/verify_inline_numbers.R`
   re-derives all **343** quoted numbers from the committed CSVs and reports
   `343 checks, 343 pass, 0 FAIL`.
2. **Regenerate every figure and table** from those CSVs (see *Regenerating the
   exhibits* below) — no cluster required.
3. **Re-run the studies from scratch** (simulation + NHANES) if desired — the full
   as-run code is here.

> **What is *not* in this repo:** the bulky per-replicate `.rds` outputs (hundreds of
> files, regenerated by re-running the studies) are not committed — only the locked
> *summary* CSVs the paper quotes. Several large studies were run on a compute
> cluster; those run packages are included exactly as run (clearly labeled), with
> any account/email/home-path scrubbed to placeholders.

## The method in one screen

The observed unit is `O = (C, A, Y)` — confounders `C`, binary treatment `A`, binary
outcome `Y` — sampled under a stratified multistage design with known design weights
`w_i = 1/π_i`, strata `h`, and primary sampling units (PSUs) `j`. The target is the
population ATE on the risk-difference scale, `Ψ = E{Y(1) − Y(0)}`. **The PSU is the
independent unit of inference**, and the asymptotics are taken as the number of
sampled PSUs grows.

**Targeting (arm-specific, two parameters).** The targeting step fits an
**arm-specific, two-parameter fluctuation `(ε₀, ε₁)`** on the two clever covariates

```
H₁(A,C) = A / g(1|C),     H₀(A,C) = (1−A) / (1 − g(1|C)),     H = H₁ − H₀
```

by the survey-weighted logistic regression of `Y` on `(H₀, H₁)` with offset
`logit Q`. This arm-specific parametrization is the one the `tmle` package fits and is
the estimator the theory certifies. (There is **no** single scalar fluctuation
parameter; `ε` is the pair `(ε₀, ε₁)`.)

**Variance.** The standard stratified **ultimate-cluster (Taylor-linearization)
estimator applied to the per-unit influence function** — accounts for weights,
stratification, and clustering at once.

**Why cross-fitting (the certified `Fully-Aware-CF`).** With flexible learners the
single-fit estimator leaves an empirical-process term that need not vanish without a
Donsker condition. Cross-fitting removes it:
- **PSU-level folds, balanced within strata**, assigned by a data-independent
  **cyclic** rule. With `mₕ = 2` PSUs (most NHANES strata) this is
  **leave-one-PSU-per-stratum** (`V = 2`); a 3-PSU stratum splits 2–1 (a bounded
  imbalance `D = o(√m)` the theory accommodates).
- **Unweighted out-of-fold nuisance fits** — under non-informative sampling given the
  confounders (`S ⫫ (A,Y) | C`) the unweighted conditional fits are consistent for
  `(Q₀, g₀)` and avoid per-fold propensity separation. Out-of-fold propensities are
  truncated to `[0.05, 0.95]`, outcome predictions to `[1e−3, 1−1e−3]`.
- **Pooled weighted targeting, then the design variance** — the survey weights
  re-enter at the pooled `(ε₀, ε₁)` targeting and at the variance.

**Sampling assumption.** Design weights are **deterministically bounded** relative to
the population-average weight (`maxᵢ wᵢ ≤ κ N / m`); a uniform lower bound on
inclusion probabilities is deliberately **not** assumed.

### The five comparison arms (`codes/estimators.R`)

| Arm | Nuisances | Cross-fitting | Variance | Role |
|---|---|---|---|---|
| **Non-Aware** | unweighted | none | i.i.d. | ignores the design |
| **Partially-Aware** | weighted | none | weighted, **no** clustering | weights but no design variance |
| **Fully-Aware** | weighted (single fit) | none | design-based | valid only under a Donsker restriction |
| **Fully-Aware-CV** | internal cluster-aware CV | none (ensemble selection only) | design-based | shows internal CV ≠ cross-fitting |
| **Fully-Aware-CF** | **unweighted out-of-fold** | **PSU-level** | design-based | **certified primary estimator** |

The internal run IDs (R01…R12) are kept so they cross-reference the run notes and the

| **R02** largem_sweep | single-fit failure worsens as PSUs/stratum grow | `R02_largem_sweep_summary.csv` | Web App F (`tab:webF_largem`) |
| **R03** isolation_2x2 | cross-fitting vs de-weighting, isolated 2×2 | `R03_isolation_2x2_summary.csv` | Web App D (`tab:webD_iso`) |

| **R07** informative | robustness to informative selection | `R07_informative_summary.csv` | Web App F (`tab:webF_informative`) |

## How to use this repository

From the repository root, after a one-time `Rscript install_packages.R`:

### 1. Verify every number in the paper (fast; no re-running)

```bash
Rscript codes/verify_inline_numbers.R     # -> "343 checks, 343 pass, 0 FAIL"
```

### 2. Regenerate the figures and tables (from the committed CSVs)

```bash
Rscript codes/make_figures.R              # Figure 1 + Table 1
Rscript codes/render_mi_tex.R             # Web App F MI table   (and the other render_*.R)
Rscript Nhanes/R/render_table2_tex.R      # Main Table 2 (five-arm, MI m=40)
```

Renderers write LaTeX fragments into `Writing/Appendix/` and `Writing/manuscript/`
and figures into `Writing/manuscript/images/`.

### 3. Re-run the simulation study

```bash
Rscript codes/run_smoke.R                 # small local smoke (minutes)
# full study (1000 reps, L1-L4 ladder x 2 designs) — run on a cluster:
#   sbatch codes/sim.slurm                # then: Rscript codes/aggregate_sim.R
```

The complexity ladder (`codes/learners.R`) runs from `L1` (parametric, single-fit
valid) to `L4` (deep random forest, single-fit under-covers, cross-fitting recovers).

### 4. Re-run the NHANES analyses

```bash
Rscript Nhanes/R/00_download_cache.R      # downloads + caches raw NHANES (internet)
Rscript Nhanes/R/01_build_analytic.R      # build E1-E4 analytic datasets
Rscript Nhanes/R/02_impute.R              # impute covariates
Rscript Nhanes/R/03_run_estimators.R      # the five arms per example
Rscript Nhanes/R/04_descriptives.R        # Web App E descriptives
Rscript Nhanes/R/05_figures.R             # Figure 2 (forest plot)
```

The headline Table 2 uses **multiple imputation (m = 40)**: that is the
`Nhanes/arc_runs/R06_mi` run package; the four examples are defined in
`Nhanes/R/examples.R`:

| id | Illustration | Cycles |
|---|---|---|
| **E1** | short sleep → obesity | 2007–2018 |
| **E2** | food insecurity → depression (PHQ-9 ≥ 10) | 2007–2018 |
| **E3** | e-cigarette use → hypertension | 2015–2018 |
| **E4** | history of gestational diabetes → later-life hypertension | 2007–2018 |

## Dependencies

R (≥ 4.2). Install with `Rscript install_packages.R`:
`SuperLearner`, `tmle`, `survey`, `surveyCV`, `earth`, `gam`, `glmnet`, `ranger`
(engine + simulation ladder); `mice`, `dplyr`, `tidyr`, `purrr`, `nhanesA`
(NHANES build / imputation / download); `rsimsum` (optional simulation summaries).
`statmod` is **not** required — Gauss–Hermite integration is computed in base R in
`codes/dgp.R`.
