## What this repository contains

The **complete analysis code** for the paper, the **locked summary-result files**
every quoted number is computed from, and a **number audit** that re-derives every
figure/table/inline number from those files:

- **`R/`** — the reusable core: the five-arm estimator engine plus the
  data-generating process, the Super Learner complexity ladder, and shared config.
- **`simulation/`** — the simulation study (drivers, renderers, the eight
  enhancement/sensitivity simulations, and the cluster scripts).
- **`nhanes/`** — the four-example NHANES analysis (download → build → impute →
  estimate → figures) plus the three NHANES sensitivity analyses (including the
  multiple-imputation run behind the headline Table 2).
- **`results/`** — the locked summary CSVs the paper quotes (small; committed).
- **`verify_numbers.R`** — re-derives **all 460** quoted numbers from `results/`.
- **`outputs/`** — where generated tables/figures land (gitignored).

From a fresh clone, after `Rscript install_packages.R`, a reviewer can:

1. **Verify every number in the paper** — `Rscript verify_numbers.R` →
   `460 checks, 460 pass, 0 FAIL`.
2. **Regenerate every figure and table** from the committed CSVs (see below) — no
   cluster required.
3. **Re-run the studies from scratch** (simulation + NHANES) — the full as-run code
   is here.

> **What is *not* committed:** the bulky per-replicate `.rds` outputs (regenerated by
> re-running the studies) — only the locked *summary* CSVs the paper quotes. The
> large studies were run on a compute cluster; those run packages are included
> exactly as run (clearly labeled), with any account/email/home-path scrubbed to
> placeholders.

## The method in one screen

The observed unit is `O = (C, A, Y)` — confounders `C`, binary treatment `A`, binary
outcome `Y` — sampled under a stratified multistage design with known design weights
`w_i = 1/π_i`, strata `h`, and primary sampling units (PSUs) `j`. The target is the
population ATE on the risk-difference scale, `Ψ = E{Y(1) − Y(0)}`. **The PSU is the
independent unit of inference**; the asymptotics are taken as the number of sampled
PSUs grows.

**Targeting (arm-specific, two parameters).** The targeting step fits an
**arm-specific, two-parameter fluctuation `(ε₀, ε₁)`** on the clever covariates

```
H₁(A,C) = A / g(1|C),     H₀(A,C) = (1−A) / (1 − g(1|C)),     H = H₁ − H₀
```

by the survey-weighted logistic regression of `Y` on `(H₀, H₁)` with offset
`logit Q`. This is the parametrization the `tmle` package fits and that the theory
covers. (There is **no** single scalar fluctuation parameter; `ε` is the pair
`(ε₀, ε₁)`.)

**Variance.** The standard stratified **ultimate-cluster (Taylor-linearization)
estimator applied to the per-unit influence function** — weights, stratification, and
clustering at once.

**Why cross-fitting (the primary `Fully-Aware-CF`).** With flexible learners the
single-fit estimator leaves an empirical-process term that need not vanish without a
Donsker condition. Cross-fitting removes it:
- **PSU-level folds, balanced within strata.** With `mₕ = 2` PSUs (most NHANES
  strata) this is **leave-one-PSU-per-stratum** (`V = 2`); a 3-PSU stratum splits 2–1
  (a bounded imbalance `D = o(√m)` the theory accommodates).
- **Unweighted out-of-fold nuisance fits** — valid under non-informative sampling
  given the confounders (`S ⫫ (A,Y) | C`); avoids per-fold propensity separation.
  Out-of-fold propensities are truncated to `[0.05, 0.95]`, outcomes to `[1e−3, 1−1e−3]`.
- **Pooled weighted targeting, then the design variance** — the weights re-enter at
  the pooled `(ε₀, ε₁)` targeting and at the variance.

**Sampling assumption.** Design weights are **deterministically bounded** relative to
the population-average weight (`maxᵢ wᵢ ≤ κ N / m`); a uniform lower bound on
inclusion probabilities is deliberately **not** assumed.

### The five comparison arms (`R/estimators.R`)

| Arm | Nuisances | Cross-fitting | Variance | Role |
|---|---|---|---|---|
| **Non-Aware** | unweighted | none | i.i.d. | ignores the design |
| **Partially-Aware** | weighted | none | weighted, **no** clustering | weights but no design variance |
| **Fully-Aware** | weighted (single fit) | none | design-based | valid only under a Donsker restriction |
| **Fully-Aware-CV** | internal cluster-aware CV | none (ensemble selection only) | design-based | shows internal CV ≠ cross-fitting |
| **Fully-Aware-CF** | **unweighted out-of-fold** | **PSU-level** | design-based | **primary estimator** |

## How to use this repository

Run everything **from the repository root**, after a one-time `Rscript install_packages.R`.

### 1. Verify every number in the paper (fast; no re-running)

```bash
Rscript verify_numbers.R                  # -> "460 checks, 460 pass, 0 FAIL"
```

### 2. Regenerate the figures and tables (from the committed CSVs)

```bash
Rscript simulation/make_figures.R         # Figure 1 + Table 1
Rscript simulation/renderers/render_webD_tex.R     # (and the other simulation/renderers/*.R)
Rscript nhanes/renderers/render_table2_tex.R       # Main Table 2 (five-arm, MI m=40)
```

Renderers write LaTeX fragments into `outputs/tables/` and figures into
`outputs/figures/`. (The Web-Appendix-E descriptive tables,
`nhanes/renderers/render_table1_tex.R`, need the NHANES analytic data — run the
pipeline below first.)

### 3. Re-run the simulation study

```bash
Rscript simulation/run_smoke.R            # small local smoke (minutes)
# full study (1000 reps, L1-L4 ladder x 2 designs) — on a cluster:
#   sbatch simulation/slurm/sim.slurm ; then: Rscript simulation/aggregate.R
```

### 4. Re-run the NHANES analyses

```bash
Rscript nhanes/00_download_cache.R        # downloads + caches raw NHANES (internet)
Rscript nhanes/01_build_analytic.R        # build E1-E4 analytic datasets
Rscript nhanes/02_impute.R                # impute covariates
Rscript nhanes/03_run_estimators.R        # the five arms per example
Rscript nhanes/04_descriptives.R          # Web App E descriptives
Rscript nhanes/05_figures.R               # Figure 2 (forest plot)
```

The headline Table 2 uses **multiple imputation (m = 40)** —
`nhanes/sensitivity/R06_mi`. The four examples (`nhanes/examples.R`):

| id | Illustration | Cycles |
|---|---|---|
| **E1** | short sleep → obesity | 2007–2018 |
| **E2** | food insecurity → depression (PHQ-9 ≥ 10) | 2007–2018 |
| **E3** | e-cigarette use → hypertension | 2015–2018 |
| **E4** | history of gestational diabetes → later-life hypertension | 2007–2018 |

## Dependencies

R (≥ 4.2). Install with `Rscript install_packages.R`:
`SuperLearner`, `tmle`, `survey`, `surveyCV`, `earth`, `gam`, `glmnet`, `ranger`
(engine + simulation ladder); `mice`, `dplyr`, `tidyr`, `purrr`, `nhanesA`

required — Gauss–Hermite integration is computed in base R in `R/dgp.R`.

## Engine note (2026-07)

Two fixes were applied to `R/estimators.R` after an adversarial pre-submission review:

1. **`cvQinit = FALSE` is now explicit** on the single-fit (Fully-Aware / Non-Aware) `tmle()`
   calls. `tmle` >= 2.x silently defaults `cvQinit = TRUE` (a 10-fold cross-validated *initial
   outcome fit*), so the "single-fit" arms were previously not single-fit on the outcome side.
   The direction of the old default was *conservative* for the paper's message: it made the
   single-fit comparator harder to break, yet it broke anyway.
2. **The EIF reconstruction (`.eif_from_tmle`) now uses the fit's own g-bounds** (`fit$g$bound`),
   replicating `tmle`'s internal per-arm bounding exactly, instead of a hard-coded `1e-3`
   truncation that could differ from what the targeting step solved.

The `J3`/`J4`/`J5` enhancement runs (and their locked summaries) were produced under this fixed
engine. The headline `sim_full_summary.csv` and NHANES results are being regenerated under the
same engine; until that lands, re-running the headline scripts against the current `R/` will
reproduce the qualitative findings with small numeric shifts concentrated in the L3/L4
single-fit cells (single-fit coverage the same or lower).

## Reproducibility

The analyses were run on a compute cluster under **R 4.4.0**; figures, tables, and the
number audit are regenerated locally under **R 4.5.1**. The exact package versions for
both environments are in [`SESSIONINFO.md`](SESSIONINFO.md), and the per-run versions
are recorded in each run's manifest. Every quoted number is re-derived from the
committed summary CSVs by `Rscript verify_numbers.R` (**460 checks, 460 pass**), so a
reviewer can confirm the results without re-running the studies.
